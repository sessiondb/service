# Single config.toml (scli + backend)

## Purpose

One TOML file holds all configuration for SessionDB: server, database, Redis, JWT, auth (default logins), and UI (API URL). **scli init** creates it; the backend and scli both read from it so you don’t maintain separate .env and config files per environment.

## Where it lives

- **Path:** `config.toml` inside the config directory.
- **Config directory:** `$HOME/.config/sessiondb` (or `SESSIONDB_CONFIG_DIR`), or `/opt/sessiondb` when running as root.
- **Override:** Set `CONFIG_TOML_PATH` to the full path of the file if you use a different location.

## Structure (scli init standard)

```toml
[server]
port = "8080"
mode = "production"

[database]
host = "localhost"
port = "5432"
user = "sessiondb"
password = "..."
name = "sessiondb"
ssl_mode = "disable"

[redis]
addr = "localhost:6379"
password = ""
db = 0

[jwt]
secret = "..."
expiry_hours = 24
refresh_expiry = 720

[secrets]
db_credential_encryption_key = "..."
migrate_token = "..."

[auth]
[[auth.default_logins]]
email = "admin@sessiondb.internal"
password = "admin123"
role_key = "super_admin"

[[auth.default_logins]]
email = "guest@sessiondb.internal"
password = "guest123"
role_key = "analyst"

[ui]
api_url = "http://localhost:8080/v1"
```

- **server** — API server port and mode (env vars still override when set).
- **database** — PostgreSQL connection (maps to DB_* env).
- **redis** — Redis connection (REDIS_*).
- **jwt** — JWT secret and expiry (JWT_*).
- **secrets** — Generated by `scli init`: encryption key and migrate token (sensitive; file should be 0600).
- **auth** — Default platform logins seeded once when no users exist (see [Default logins](default-logins.md)).
- **ui** — `api_url` is passed as `API_URL` to the UI server binary so the frontend points to the correct API.

## Who uses it

| Consumer | How |
|----------|-----|
| **scli** | `scli init` writes config.toml; run/start/deploy read it and build env (and generate .env for systemd). |
| **Backend (sessiondb-server)** | When `CONFIG_TOML_PATH` or `SESSIONDB_CONFIG_DIR` is set, loads config from that config.toml; env vars override. |
| **UI (sessiondb-ui)** | Gets `API_URL` from env; scli sets it from config.toml when starting the UI process. |

## Backward compatibility

- If **config.toml** is missing, scli falls back to `.env` then `config.yaml`.
- Backend falls back to `.env` and optional `config.default.toml` for default logins only when no single config.toml is found.
- **Deploy** still uses `EnvironmentFile=.env` in the systemd unit; scli generates that .env from config.toml when needed.
