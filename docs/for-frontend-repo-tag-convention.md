# For frontend repo: tag convention for binders

Add this to your frontend repo’s `docs/README.md` (or equivalent):

---

**Versioned releases (binders)**  
Use plain version tags (e.g. `1.0.1`) so they match the backend and the binders release workflow. When the binders repo runs on tag `1.0.1`, it clones the frontend repo at that tag to build the UI bundle.

---
